# 第三方模組接入前先依實際介面驗證 import 與主要入口

## 核心要求

- 新增或改寫第三方模組接入時，先使用 `$find-docs` skill 查該模組的官方文件或官方 repo docs。
- 在目前 runtime 與模組系統下，先確認套件實際 export shape 與主要入口，再進行正式整合。
- 至少做一個最小 smoke check，驗證 import 形狀、constructor 或主要呼叫入口可正常取得。
- 規則描述優先寫成正向要求，例如先驗證模組介面並用最小案例確認可用，而不是只記錄某一次錯誤訊息。

## 驗證方式

- 檢查新增或改寫的第三方模組接入，是否有對應的最小 smoke check、測試或明確驗證步驟。
- 檢查接入前是否有先使用 `$find-docs` 查官方文件或官方 repo docs，而不是只憑名稱或二手範例推測。
- 檢查實作是否依目前 runtime 的實際 export shape 接線，而不是只依名稱推測。
- 檢查規則文字是否描述正向做法，例如驗證 import/constructor/主要入口，而非僅列出禁止案例。
