# 區分新的規則建議與既定流程狀態說明

## 核心要求

- 回報中的「規則建議」區塊只應放這一輪新收斂出的治理改善、加嚴建議，或尚未落地的後續規則項。
- 已存在的 baseline 同步機制、既定 review 流程、既有規則的狀態說明，應視為流程狀態或續作脈絡，不應包裝成新的規則建議。
- 若某段內容只是說明下次從哪個 baseline 開始、哪個 topic 已完成或其他正常續作方式，應放在一般進度說明，而不是規則建議區塊。
- 只有當流程本身被修改、加嚴、放寬，或新增新的治理邊界時，才應提升為規則建議或 follow-up topic。

## 驗證方式

- 檢查規則建議區塊是否只保留新的治理改善、加嚴建議或尚未落地的後續規則項。
- 檢查既定 baseline mechanics、next-review flow 與已完成 topic，是否被寫成一般狀態說明而非新規則建議。
- 檢查 shared baseline review 的 topic 清單，是否聚焦在真正需要本地判斷與採納的差異，而不是重述既有流程。
